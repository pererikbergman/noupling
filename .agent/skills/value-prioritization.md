# Value-Prioritization Skill

- **Protocol:**
  1. **Must-Have:** Analyze if the core app flow breaks without it or if a critical legal/safety requirement is missing.
  2. **Should-Have:** Analyze if user friction is high without it, but a workaround exists.
  3. **Could-Have:** Analyze if it's purely a "Delighter" or secondary value feature.
  4. Always weight **Business Impact** (User Reach x Revenue Potential) against **Implementation Cost**.
