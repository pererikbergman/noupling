Most linters check style.
This one checks architecture.

noupling scans your project, builds a dependency graph from actual import statements, and gives you a single health score you can track over time and gate in CI.

Get Started View on GitHub 
# Install
brew tap pererikbergman/noupling
brew install noupling

# Scan, audit, report
noupling scan .
noupling audit . --fail-below 80
noupling report . --format html

Architecture, quantified

🔍

Detect coupling violations

Sibling modules that depend on each other break architectural boundaries. noupling finds every one, scores severity by depth, and tells you exactly which import to fix.

🔄

Find circular dependencies

Dependency chains that form loops prevent independent development. The XS metric identifies the weakest link in every cycle—the cheapest edge to cut.

📈

One health score (0–100)

Track architectural drift across snapshots. Gate CI with --fail-below. Drill into interactive HTML reports to see exactly where coupling lives.

⚡

14 languages, one tool

Tree-sitter parses actual ASTs—no regex. From Kotlin to Rust to Python, the same analysis runs everywhere. Rayon parallelizes scanning across all cores.

🛠

Configurable layers & rules

Define architectural layers so downward dependencies aren't flagged. Add custom rules to forbid specific import patterns. Suppress known violations inline.

📦

8 report formats

JSON, XML, Markdown, interactive HTML, SonarCloud, Mermaid, GraphViz DOT, and zoomable Sunburst. Integrate with any CI system or documentation pipeline.
